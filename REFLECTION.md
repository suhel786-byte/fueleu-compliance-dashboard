# Reflection on Using AI Agents

## What I Learned Using AI Agents

During the development of this project, AI tools played a significant role in accelerating the development process. I learned how AI agents can assist in generating boilerplate code, explaining complex programming concepts, and helping debug errors efficiently. Using AI also improved my understanding of architectural patterns such as hexagonal architecture and how to structure backend services properly.

Additionally, AI tools helped in writing documentation, generating API structures, and suggesting improvements to the codebase. This experience demonstrated how AI can act as a collaborative assistant during software development.



## Efficiency Gains vs Manual Coding

Using AI agents significantly improved productivity compared to writing everything manually. Tasks such as creating route handlers, generating TypeScript structures, and writing documentation were completed much faster.

AI tools were particularly helpful for:

* Generating initial code templates
* Debugging runtime or syntax errors
* Explaining architectural design patterns
* Improving documentation quality

However, AI outputs were not always perfect and required validation and manual adjustments to ensure correctness.



## Improvements for Future Development

In future projects, I would focus on integrating AI tools earlier in the development lifecycle and using them more strategically for architecture planning and testing automation.

I would also combine multiple AI tools to leverage their strengths—for example, using one tool for code generation and another for code review or refactoring. Additionally, I would place more emphasis on verifying AI-generated outputs to avoid potential inaccuracies.

Overall, AI agents proved to be valuable tools for improving development efficiency while still requiring human oversight and decision-making.
